## 6. Shared post form — see `03-admin-dashboard.md` ("The shared post form")
- [x] Top-level fields (title, short description, featured image, written by, status, date, featured toggle, long description)
- [x] Section builder: add/remove section
- [x] Section builder: add/remove paragraph block within a section
- [x] Section builder: add/remove image block within a section
- [x] Quote section type
- [x] Save wired to nested Prisma create/update (Section + SectionBlock)

Notes:
- Sub-task 1 (top-level fields) done. Fields render in the exact spec order: title, short description, featured image, written by, category, status, publish date, featured checkbox, long description.
- Image upload uses server-side signed Cloudinary upload: `POST /api/admin/upload` (session-protected, 10MB cap, image-type check) → returns secure_url → DB stores only the URL. Reusable `ImageUpload` component with preview/replace/remove; `compact` variant prepared for section image blocks.
- One `PostForm` client component serves both create + edit (via `mode` prop). `/admin/posts/new?category=X` pre-fills + locks category; `/admin/posts/[id]/edit` loads existing data with `lockCategory={false}` (spec allows category change as a deliberate edit-mode exception).
- Full `PostFormValues` state shape (incl. sections array) is set up now so later sub-tasks slot in without refactoring. Submit currently validates required fields then shows a "save not wired yet" message — Prisma save is sub-task 4.
- Sub-task 2 (section builder) done. "Page sections" area below the top-level card with an "Add section" button (the "Add quote" button is sub-task 3).
- Each CONTENT section has: a heading input, an ordered stack of blocks, "Add paragraph" / "Add image" buttons at the bottom, plus a "Remove section" button. Sections are numbered "Section N · Content" by their array position.
- Blocks are a single ordered list per section, so paragraphs and images interleave in any order. Each block has its own Delete button. Image blocks reuse `ImageUpload` (compact variant). Paragraph blocks are textareas.
- Block/section identity uses `crypto.randomUUID()` keys client-side (kept separate from DB ids); the DB `order` is derived from array index at save time (sub-task 4).
- Sub-task 3 (quote section) done. "Add quote" button sits next to "Add section". Quote sections live in the same `sections` array as content sections, so a quote can appear before/after/between content sections (order = array position).
- Quote section is visually distinct: cream `bg-primarybg` with a left `border-yellow` accent bar + quote icon, vs the white content-section cards. Two fields only — quote text (required textarea) + attribution (optional input). No heading, no blocks.
- Sub-task 4 (save wired to Prisma) done. `savePostAction` server action handles both create and edit; the `PostForm` submit calls it directly.
- Section `order` and block `order` are both derived from array index, so the form's visual order is the persisted order.
- Edit mode uses the delete-and-rewrite approach (inside a `prisma.$transaction`): `section.deleteMany({ postId })` cascades to blocks, then the post is updated with a fresh nested `sections.create`. Avoids diffing entirely.
- Slug: generated once on create via `lib/slug.ts` (`generateUniqueSlug` — slugify + random suffix on collision). Edit mode intentionally does NOT change the slug, per spec ("slugs should not change once published").
- CONTENT sections save heading + ordered SectionBlock rows; QUOTE sections save only quoteText + attribution (attribution stored as null when blank), no blocks.
- Server re-validates all required fields (incl. per-section/per-block) authoritatively; client validation is just a fast first pass.
- On success: `revalidatePath` on the relevant admin list + `redirect` back to it.
- Decision not in spec: server-side validation rejects empty paragraph blocks / un-uploaded image blocks with a specific "Section N, block M" message, rather than silently saving empty blocks.
- Verified end-to-end with a throwaway script: created a full Insight (content section paragraph→image→paragraph + a quote section), read it back ordered (order preserved exactly), confirmed it showed in the `/admin/insights` query, then cascade-deleted it. Script removed after verifying.
